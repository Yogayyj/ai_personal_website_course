第5课：网络基础与 GitHub Pages 站点类型

这节课帮助学生建立最关键的网络认知：IP、网址、域名是什么，以及 GitHub Pages 的 User Site 和 Project Site 有什么区别。

本节课学习目标

  • 理解 IP、URL（网址）、域名的基本概念与关系。
  • 理解 GitHub Pages 的两种网站类型：User Site 与 Project Site。
  • 能够根据用途选择正确仓库名并推导最终访问链接。
  • 避免常见部署错误（仓库名错误、路径错误、缺少 index.html）。

网络基础速记

  • IP 地址：服务器在网络中的数字地址（像门牌号）。
  • 域名：给 IP 起的“人类可读名字”（如 github.io、youjia.dev）。
  • URL 网址：完整访问地址（协议 + 域名 + 路径），例如 https://yogayyj.github.io/my-game。

一句话区别（课堂必须记住）

User Site 是你的 GitHub 账号主站：https://用户名.github.io
Project Site 是某个仓库的网站：https://用户名.github.io/仓库名

1) 仓库名不同

User Site：仓库名必须严格等于 用户名.github.io。
例如用户名是 yogayyj，仓库名必须是 yogayyj.github.io。

Project Site：仓库名可以自由命名，例如 my-game、personal-website、ai-demo。

2) 网站地址不同

类型仓库名网站地址
User Siteyogayyj.github.iohttps://yogayyj.github.io
Project Sitemy-gamehttps://yogayyj.github.io/my-game
Project Siteportfoliohttps://yogayyj.github.io/portfolio

核心差异：User Site 没有项目路径；Project Site 会多一段 /仓库名。

3) 用途不同

  • User Site 适合：个人主页、简历站、作品集首页、长期个人品牌网站。
  • Project Site 适合：小游戏、课堂作业、AI Demo、实验网页、单独项目展示。

4) GitHub Pages 设置位置（两者相同）

仓库 → Settings → Pages
Source: Deploy from a branch
Branch: main
Folder: / (root)
Save

结论：开启步骤几乎一样，真正最大区别在仓库名与最终链接。

5) 关键对比表（考试版）

对比User Site普通项目网站
仓库名要求必须是 用户名.github.io任意命名
网站链接https://用户名.github.iohttps://用户名.github.io/仓库名
适合内容主个人网站单个项目展示

6) index.html 要求（两者都一样）

首页文件建议命名为 index.html，并放在 Pages 选择的目录中。若选择 / (root)，则应放在仓库根目录。

仓库
├── index.html
├── avatar.jpg
└── README.md

缺少 index.html 可能导致 404 或默认页面。

7) 自定义域名差异

  • User Site：更适合绑定主域名（如 youjia.dev）。
  • Project Site：更常绑定子域名（如 game.youjia.dev），也可以保持默认 GitHub 链接。

8) 教学话术（可直接对学生说）

User Site 是你的“个人官网”，仓库名必须是 用户名.github.io。
Project Site 是你的“项目页面”，仓库名可自定义，但链接后面会带 /仓库名。

9) 举例说明

假设用户名是 student123：

  • 个人主页：student123.github.io → https://student123.github.io
  • 小游戏：jump-game → https://student123.github.io/jump-game
  • AI Demo：ai-chat-demo → https://student123.github.io/ai-chat-demo

最终记忆版

用户名.github.io
= GitHub 账号的主网站
= https://用户名.github.io

普通仓库名
= 某个项目的网站
= https://用户名.github.io/仓库名

最核心区别：User Site 看用户名，Project Site 看仓库名。
