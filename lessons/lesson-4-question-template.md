# Lesson 4 提问模板（先给上下文，再提具体任务）

> 不要只问：
>
> **“帮我做一个网站”**
>
> 这样太宽泛，AI 不知道你的教学对象、课程目标、学生水平、项目阶段。

## 推荐提问模式

你现在是我的课程项目助教。  
请先阅读下面的 `PROJECT_CONTEXT.md`。  
我的目标是推进“三节课个人网站项目”。  
请基于当前项目阶段，帮我完成【具体任务】。  
输出要求是【具体格式】。

---

## 可直接复制的提问模板

```markdown
你现在是我的课程项目助教。
请阅读 PROJECT_CONTEXT.md。
我现在要完成【填写你的任务，例如：第二节课讲义 / 首页文案 / 部署检查清单】。
请基于当前项目阶段输出：
1. 【你需要的内容1】
2. 【你需要的内容2】
3. 【你需要的内容3】
输出格式要求：
- 风格：适合【初学者 / 中学生 / 大学生】
- 长度：约【X】字
- 结构：使用【标题 + 列表 / 表格 / 分步骤】
```

---

## 示例（教学讲义场景）

你现在是我的课程项目助教。  
请阅读 `PROJECT_CONTEXT.md`。  
我现在要准备第二节课的课堂讲义，主题是 GitHub Pages 部署个人网站。  
请帮我生成一份适合初学者的教学讲义，要求包含：

1. 教学目标
2. 操作步骤
3. 常见错误
4. 课堂练习
5. 教师讲解话术

---

## PROJECT_CONTEXT.md（示例）

```markdown
## 1. 项目名称
个人网站生成与部署课程项目

## 2. 项目一句话介绍
本项目是一个面向初学者的三节课课程，目标是让学生使用 AI + GitHub Pages 制作并发布自己的第一个个人网站。

## 3. 项目目标
学生最终需要完成：
1. 规划个人网站内容
2. 使用 AI 生成个人网站 HTML 文件
3. 在 GitHub 上创建仓库
4. 使用 GitHub Pages 部署网站
5. 获得一个可公开访问的网站链接
6. 理解基础前端和网络概念
7. 能够自己修改网站文字、颜色、链接和项目内容

最终成果示例：
https://用户名.github.io
```
